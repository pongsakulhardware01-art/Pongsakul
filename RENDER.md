# วิธีการนำแอปพลิเคชันนี้ไปรันฟรีบน Render (Static Site)

แอปพลิเคชันนี้ถูกพัฒนาขึ้นด้วย **React (Vite) + Tailwind CSS** และเก็บข้อมูลในเครื่องของผู้ใช้แบบออฟไลน์ผ่านระบบ **LocalStorage** ปลอดภัย ไม่มีฐานข้อมูลหลังบ้านที่ซับซ้อน 

คุณสามารถตั้งค่ารันบน **Render** ได้ในรูปแบบ **Static Site** (ซึ่งให้บริการ **ฟรีไม่มีค่าใช้จ่าย!** และเร็วมาก) โดยทำตามขั้นตอนง่ายๆ ดังนี้ครับ:

---

## ขั้นตอนที่ 1: เตรียมโค้ดขึ้น GitHub
1. สมัครใช้งานและล็อกอินเข้าสู่ **GitHub** (https://github.com)
2. สร้าง Repository ใหม่ขึ้นมา (ติดตั้งเป็นแบบ Private หรือ Public ก็ได้)
3. อัปโหลดก้อนซอร์สโค้ดของโปรเจกต์นี้ทั้งหมด (ยกเว้นโฟลเดอร์ `node_modules` และ `dist` ซึ่งระบบปลั๊กอิน `.gitignore` จะข้ามให้อยู่แล้ว) ขึ้นไปบนระบบ GitHub ของคุณ

---

## ขั้นตอนที่ 2: ตั้งค่าบน Render (Render.com)
1. เข้าไปที่เว็บ **Render** (https://render.com) แล้วลงชื่อเข้าสู่เข้าใช้งานบัญชีของคุณ
2. ที่หน้าแดชบอร์ดหลัก กดปุ่ม **New +** ที่มุมบนขวา -> เลือก **Static Site**
3. บริการจะขอให้คุณเชื่อมโยงบัญชี GitHub -> กดอนุมัติแล้วเลือกโปรเจกต์ (Repository) ที่คุณได้พึ่งอัปโหลดขึ้นไปในขั้นตอนแรก

---

## ขั้นตอนที่ 3: ระบุข้อมูลการ Build ระบบ
เมื่อเลือกโปรเจกต์แล้ว Render จะแสดงหน้าต่างตั้งค่า ให้ใส่ข้อมูลดังนี้:

* **Name**: `company-holiday-manager` (หรือชื่อใดใดที่คุณต้องการตั้งเป็น Subdomain ฟรีของ Render)
* **Branch**: `main` (หรือชื่อกิ่งที่คุณอัปโหลดโค้ดขึ้นไป)
* **Build Command**: `npm run build`
* **Publish Directory**: `dist`

กดปุ่ม **Create Static Site** จากนั้นรอระบบ Build สักครู่ (ไม่เกิน 1-2 นาที) เมื่อเสร็จสิ้นคุณก็จะได้ลิงก์ URL (เช่น `https://company-holiday-manager.onrender.com`) เพื่อเปิดใช้งานระบบได้ทันที!

---

## 💡 คำแนะนำเพิ่มเติม (การจัดการเส้นทาง SPA):
เมื่อสับเปลี่ยนหน้าหรือหากในอนาคตมีการเพิ่มระบบ Routing (หน้าเว็บสาขาอื่น) เพื่อป้องกันอาการรีเฟรชแล้วเจอหน้า Error 404 สามารถตั้งค่าเพิ่มบน Render ได้ดังนี้:
1. ไปที่แท็บ **Redirects/Rewrites** ใน Dashboard ของแอปนั้นๆ บน Render
2. กด **Add Rule**
3. ช่อง **Source**: `/*`
4. ช่อง **Destination**: `/index.html`
5. ช่อง **Action**: `Rewrite` (เปลี่ยนจาก Redirect เป็น Rewrite)
6. กดลิสต์บันทึกเซฟข้อมูล

เพียงเท่านี้ ระบบการลาพักผ่อนและจัดการรายชื่อพนักงานของคุณก็พร้อมรันบน Render ให้ทุกคนในสำนักงานใช้งานอย่างมีมาตรฐานแล้วครับ!
